COMMON ASYNC CONTRACT

Heavy endpoints now return:
- HTTP 202 Accepted
- `task_id`
- `status`
- `queue_position`
- `poll_url`

Poll task:
GET /api/tasks/{task_id}

Task status meanings:
- `queued`
- `running`
- `completed`
- `failed`
- `cancelled`

Polling behavior:
- `202` while queued/running
- `200` once completed/failed/cancelled

Artifact endpoints:
- GET /api/tasks/{task_id}/artifacts
- GET /api/tasks/{task_id}/artifacts/{artifact_id}

Ephemeral rule:
- if `outdir` is omitted, the endpoint runs in ephemeral mode
- generated files are exposed as downloadable task artifacts
- if `outdir` is provided, output is persistent

Input-file rule:
- endpoints that support `input_file` accept a local `.txt` or `.json` file
- `.txt` expects one item per line
- `.json` expects a list of strings


TASK HELPERS

List tasks:
GET http://127.0.0.1:8000/api/tasks

Get one task:
GET http://127.0.0.1:8000/api/tasks/{task_id}

List artifacts:
GET http://127.0.0.1:8000/api/tasks/{task_id}/artifacts

Download one artifact:
GET http://127.0.0.1:8000/api/tasks/{task_id}/artifacts/{artifact_id}

Cancel queued/running task:
DELETE http://127.0.0.1:8000/api/tasks/{task_id}

SPOTIFY USER AUTH FLOW

1. Start auth
POST /api/music/user/auth/start

Response includes:
- `auth_session_id`
- `authorization_url`
- `redirect_uri`
- `poll_url`

2. Open `authorization_url` in a browser

3A. If your Spotify app redirect URI points to the API callback:
GET /api/music/user/auth/callback?code=...&state=...

3B. If you are using manual/Postman completion instead:
POST /api/music/user/auth/complete

With redirected URL:
{
  "auth_session_id": "your-auth-session-id",
  "redirected_url": "http://127.0.0.1:8000/api/music/user/auth/callback?code=...&state=..."
}

Or direct code/state:
{
  "auth_session_id": "your-auth-session-id",
  "code": "spotify_code_here",
  "state": "spotify_state_here"
}

4. Poll auth session
GET /api/music/user/auth/session/{auth_session_id}

5. Fetch playlists after auth
GET /api/music/user/playlists
or
POST /api/music/user/playlists

6. Download selected playlists
POST /api/music/user/download

Download all:
{
  "playlists": ["all"]
}

Download selected playlists:
{
  "playlists": ["Playlist A", "Playlist B"]
}

From txt/json file:
{
  "input_file": "./data/playlist_names.txt"
}

Persistent:
{
  "playlists": ["Playlist A", "Playlist B"],
  "outdir": "./downloads/spotify_user"
}
